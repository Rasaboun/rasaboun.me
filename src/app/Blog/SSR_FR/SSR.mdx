# Les avantages du rendu côté serveur (SSR) pour le développement web

### De nos jours, les sites Web doivent être rapides et optimisés pour le référencement afin de fournir la meilleure expérience utilisateur et d'améliorer le classement dans les moteurs de recherche.

### Aujourd'hui, nous allons discuter d'une solution pour obtenir un site Web rapide et optimisé pour le référencement: le rendu côté serveur (SSR).

### Mais avant d'entrer dans le vif du sujet, comprenons d'abord comment React affiche notre page Web.

---

## Rendu côté client (CSR)

Le rendu côté client est une méthode de rendu qui consiste à envoyer HTML, CSS et JavaScript au navigateur de l'utilisateur.

Une fois que tout est chargé, JavaScript prend le relais et génère la page, la page est rendue et JavaScript la rend interactive.

L'inconvénient de cette méthode est que le fichier HTML est initialement vide, et JavaScript le remplit ensuite de contenu, ce qui peut causer des problèmes de référencement.

Le robot d'exploration Web de Google peut exécuter JavaScript, ce qui permet à votre site Web d'être correctement indexé, mais cela peut prendre plus de temps car le robot doit d'abord rendre votre site.

En outre, les plateformes de médias sociaux nécessitent des balises meta pour générer un aperçu de votre site.

## Avantages

- **Temps de chargement initial plus rapide**, car le serveur envoie uniquement un fichier HTML vierge et le navigateur gère le rendu.
- **Charge serveur plus faible**, car le serveur doit uniquement envoyer des ressources statiques (HTML, CSS, JavaScript, image) au client.

## Inconvénients

- **Temps d'interaction plus lent**, car le navigateur doit d'abord charger toutes les ressources statiques, puis exécuter le code JavaScript pour rendre la page.
- **Problèmes de référencement**, les moteurs de recherche ne peuvent pas lire le contenu généré par JavaScript.
- **Temps de chargement plus long** sur les appareils ou les réseaux plus lents, car le rendu côté client nécessite plus de puissance de traitement et de mémoire.
- **Limite le nombre d'utilisateurs**, car certains utilisateurs peuvent avoir désactivé JavaScript.

---

## Parlons maintenant du Rendu Côté Serveur

Avec le rendu côté serveur, lorsqu'un utilisateur demande une page, le serveur génère la page HTML et l'envoie au navigateur, qui rend ensuite la page avec CSS et JavaScript.

Parfait, pourriez-vous dire? Eh bien, pas tout à fait!

Actuellement, notre site n'est pas interactif; par exemple, les boutons ne fonctionnent pas car l'étape d'hydratation est manquante.

*Qu'est-ce que l'hydratation?*

L'hydratation permet la transition entre la page générée par le serveur et notre navigateur.

Il permettra de lier les "**gestionnaires d'événements**" à notre page.

En bref, cela liera, par exemple, un bouton HTML à une fonction JavaScript qui effectuera une action, comme envoyer le message que vous avez écrit dans un chat.

## Avantages

- **Amélioration des performances**, car le serveur génère le HTML, CSS et JavaScript initial pour la page Web.
- **Meilleur référencement**, car les moteurs de recherche peuvent facilement explorer et indexer des pages rendues côté serveur.
- **Temps d'interaction plus rapide**, en particulier sur les appareils ou les réseaux plus lents, car le rendu initial est effectué côté serveur.
- **Amélioration de l'expérience utilisateur**, car la page Web devient interactive plus tôt.

## Inconvénients

- **Complexité accrue**, car le rendu côté serveur nécessite une configuration et une mise en place supplémentaires.
- **Charge serveur plus élevée**, car le serveur doit générer le HTML, CSS et JavaScript initial pour chaque demande.
- **Support limité pour certaines fonctionnalités côté client**, qui reposent sur le modèle d'objet de document (DOM) du navigateur.
- **Temps de chargement initial plus long**, car le serveur doit générer le HTML, CSS et JavaScript initial pour chaque demande.
- **Risque de contenu non synchronisé entre le serveur et le client**, car le serveur et le client doivent tous deux rendre le même contenu. (Hydratation)
